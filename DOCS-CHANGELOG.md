# Docs Changelog

Log of changes to this repo's own files/structure (CLAUDE.md, STATE.md, README.md, conventions). For changes that mutate the live Home Assistant instance, see `CHANGELOG.md` instead.

## 2026-06-24

1. Created `CLAUDE.md` from two prior per-machine files (`CLAUDE-nuc.md`, `CLAUDE-desktop.md`), which were identical; removed the per-machine files.
2. Added `README.md` explaining the repo's purpose and `.mcp.json` setup for new machines.
3. Added a `Process` section to `CLAUDE.md`: consult the `home-assistant-best-practices` skill before persistent config changes; log server-state mutations to `CHANGELOG.md` and commit immediately.
4. Changed `CHANGELOG.md` entries from bullets to ordered lists, to reflect logical step sequence.
5. Split point-in-time inventory (entity/device counts, areas, automations, helpers, sensors, dashboards) out of `CLAUDE.md` into a new `STATE.md`, read on demand rather than auto-loaded, so stale counts can't silently mislead. `CLAUDE.md` retained only durable conventions/policies.
6. Added this file (`DOCS-CHANGELOG.md`) to track repo/doc changes separately from HA server-state changes.
7. Added `ISSUES.md` for observations flagged during other work but not acted on (status starts `TBD`).
8. Documented in `CLAUDE.md`: always search for consumers before touching any entity/automation (no staging environment); no entity domain gets extra confirmation by default; household context (cleaners fortnightly, pet sitters during holiday mode) explaining the holiday automations' purpose.
9. Documented in `CLAUDE.md`'s Process section: ask clarifying questions one at a time rather than bundling several into one prompt.
10. Added `/state` slash command (`.claude/commands/state.md`) to revalidate `STATE.md` against the live HA instance, correct drift in place (no new sections), flag unresolved findings in `ISSUES.md`, and log the run in this changelog. Added a "Available Commands" section to `README.md`.
11. Added `/conventions` slash command (`.claude/commands/conventions.md`) to audit the live HA instance against `CLAUDE.md` conventions (naming, icons, Hue groups, entity patterns), print findings directly in chat (no report file), and log all findings to `ISSUES.md` since the audit itself doesn't action anything.
12. Added `/issue` slash command (`.claude/commands/issue.md`) to list open `ISSUES.md` entries and work through one at a time, updating its status in place once resolved. Updated `README.md`'s Available Commands list.
13. Documented in `CLAUDE.md`'s Process section: after adding a new Claude Code slash command, add it to `README.md`'s "Available Commands" list.
14. Ran `/state` to revalidate `STATE.md` against the live HA instance. Drift found and corrected: entity count 482→554, automation count 58→67; moved Man Cave from "Garage/External" to "House" (matches live floor data); Room automations 24→28 (added Man Cave Sim Rig On/Off, Kitchen Light Bar On With No Motion/Turned Off); Routine 9→8 (removed `Routine: Bedtime Start`, no longer found live); Alert 6→8 (added Meater Cook Started, Meater Estimated Time Available); System 1→2 (added IKEA Hub Auto-Restart); added `input_boolean.ben_s_ipad_in_use` to Helpers; corrected Key Devices' Main TV entry (`media_player.living_room_tv_2` no longer exists). 5 ambiguous findings (uncategorized `Holiday:`-prefixed automations, `Garage override:` naming, unverifiable "Pool area" sensor, missing `Routine: Bedtime Start`, ambiguous Main TV replacement) logged to `ISSUES.md` rather than guessed at.
15. Renamed the `/issue` slash command to `/issues` (`.claude/commands/issue.md` → `.claude/commands/issues.md`) and updated `README.md`'s Available Commands list.
16. Added `/rename` slash command (`.claude/commands/rename.md`) to propose a better name for a poorly-named device/entity, search for all consumers (automations, scripts, scenes, dashboards) per `CLAUDE.md`'s consumer-check rule, and — once the user approves the proposed name — apply the rename live and patch every consumer found, logging the change to `CHANGELOG.md`. Updated `README.md`'s Available Commands list.
17. Extended `CLAUDE.md`'s Naming Convention section: added a device-naming convention (`{Area} {Fixture}` title case, mirroring entity_id) so vendor-default device names can be replaced consistently; added an exceptions clause for non-area-bound entities/devices (`person.*`, `zone.*`, `update.*`, diagnostics). Extended Common Entity Patterns with `switch`, `lock`, `media_player`, `fan`, `button`, `vacuum`, and `camera` domain patterns (previously only `light`/`climate`/`cover`/`sensor` were covered).

## 2026-06-25

1. Added the `ha-verify` skill (`.claude/skills/ha-verify/SKILL.md`): a repeatable test loop — propose acceptance criteria, set up baseline state via service calls, trigger (Claude for HA-internal state, the user for real-world events), assert the outcome via live MCP queries, and on failure fix-and-retest in place rather than auto-rolling-back. Added a thin `/verify` slash command wrapper (`.claude/commands/verify.md`). Documented in `CLAUDE.md`'s Process section that every persistent config change must pass this loop before the changelog/commit step; the same skill also applies standalone for debugging, with no changelog step in that case. Updated `README.md`'s Available Commands list and added a new Skills section.
2. Added `IDEAS.md` for speculative, unscheduled ideas bigger than an `ISSUES.md` entry. First entry: building a true offline unit-test harness for automations (exporting them to YAML + `pytest-homeassistant-custom-component`), revisited if automation complexity grows or this tooling gets open-sourced. Added it to `README.md`'s Contents list.
3. Added `VERIFY.md` for `ha-verify` checks that couldn't be confirmed live (time-of-day/presence/physical-trigger gated) — each entry records what's already been checked, why the live test is blocked, and acceptance criteria for when to retry. Updated the `ha-verify` skill (`.claude/skills/ha-verify/SKILL.md`) with a "time-sensitive test" workflow: verify whatever's checkable now, log the rest to `VERIFY.md` instead of dropping it. Updated `/verify` (`.claude/commands/verify.md`) so that with no argument it works through `VERIFY.md`'s `Pending` entries instead of asking what to test. Added `VERIFY.md` to `README.md`'s Contents list.
4. Scrubbed personal names from framing prose in `README.md`, `STATE.md`, `CHANGELOG.md`, and `CLAUDE.md` ahead of making the repo public — replaced with generic phrasing (e.g. "this Home Assistant instance", "residents"). Left literal entity_ids and live automation alias examples untouched, since those are factual records of this specific install rather than promotional text, and genericizing them would make the docs stop matching the live system.
5. Added a `CLAUDE.md` Process bullet: avoid writing personal names into committed files going forward (use "primary user"/"partner" or "[redacted]"), with the same entity_id/automation-alias exemption as above, so this doesn't need to be re-scrubbed later.
6. Added `/new-setup` slash command (`.claude/commands/new-setup.md`) to reset the instance-specific docs (`STATE.md`, `CHANGELOG.md`, `DOCS-CHANGELOG.md`, `ISSUES.md`, `IDEAS.md`, `VERIFY.md`) to empty skeletons, for starting a fresh fork of this repo. Added a "Using this as a template" section to `README.md` (clone → drop git history → set up `.mcp.json` → run `/new-setup` → run `/state`) and updated the Available Commands list.
7. Split `CLAUDE.md` into three files by concern: `CLAUDE.md` keeps process/policy/household-context/automation-patterns; new `CONVENTIONS.md` holds opinionated naming/icon/entity-pattern conventions (Naming Convention, Icon Guidelines, Common Entity Patterns); new `GOTCHAS.md` holds surprising technical quirks (Hue Groups sync issues, the Google Cast TTS/Spotify session teardown quirk). Updated cross-references in `/conventions`, `/rename`, `ISSUES.md`, and `STATE.md` to point at the new files. Added both to `README.md`'s Contents list.
8. Added a `Makefile` (`make lint-md`) and `.markdownlint-cli2.jsonc` (modeled on benrowe/freighter-down's config) to lint all root-level `.md` files; `.claude/**` is excluded since slash-command/skill files intentionally lack a top-level heading. Disabled rules that fight this repo's established style (line-length, blanks-around-headings/lists, table column alignment, fenced-code-language, emphasis-as-heading) rather than reformatting everything; fixed the handful of genuine issues found (a heading-level skip in `CONVENTIONS.md`, a missing blank line around a fenced code block in `README.md`, blank lines inside a blockquote in `STATE.md`). Documented in `CLAUDE.md`'s Process section: run `make lint-md` before committing any `.md` change. Updated `README.md`'s Contents list.
