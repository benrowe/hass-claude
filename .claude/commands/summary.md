3. Present both lists together, then suggest 1-2 items worth tackling next based on impact/effort (e.g. quick wins from `ISSUES.md`, or an idea whose trigger condition in `IDEAS.md` now seems to apply).
4. Ask the user if they want to act on anything — if `ISSUES.md`, hand off to `/issues`; if `IDEAS.md`, discuss directly per `CLAUDE.md`'s Process section (propose, wait for confirmation before acting).
